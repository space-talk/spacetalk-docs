# Space Object Physical Characteristics

## Purpose and scope
This optional block describes mass properties, dimensions, cross‑section areas for drag/SRP, radar/optical observables, attitude-control descriptors, and inertia. It supports perturbation modeling and collision-probability estimation. At most one PHYS block may appear in an OCM.

## Block keywords and order
Delimited by:
- PHYS_START (line by itself)
- PHYS_STOP (line by itself)

Within the block, KVN keys shall appear in the fixed order below.

## Validation and constraints

- Only one PHYS block per OCM.
- If any OEB_Q* is present, OEB_PARENT_FRAME must be supplied (and epoch if required).
- Quaternion values of −999 denote tumbling; treat OEB orientation as undefined in that case.
- If CR or CD is provided as 0, ignore the corresponding non-gravitational force in modeling (per standard guidance).
