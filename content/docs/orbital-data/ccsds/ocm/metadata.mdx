# OCM Metadata

## Purpose and scope
The Metadata block declares identification, timing, time system, and optional provenance and linkage needed to interpret all subsequent data blocks. Exactly one metadata block shall be present.

## Block keywords and order
Delimited by:
- META_START (line by itself)
- META_STOP (line by itself)

All KVN assignments between these delimiters shall appear in the fixed order given below. Only one metadata block is allowed in an OCM.

## Validation rules and constraints
- Exactly one metadata block per OCM.
- TIME_SYSTEM and EPOCH_TZERO are mandatory and must precede any relative-time sections.
- SCLK usage requires both SCLK_OFFSET_AT_EPOCH and SCLK_SEC_PER_SI_SEC; interpret EPOCH_TZERO in UTC in that case.
- START_TIME/STOP_TIME, when present, may be absolute or relative; do not mix within the same KVN assignment, and ensure consistency with the chosen interpretation of the blocks.
- All absolute time strings must validate against CCSDS formats (see /ocm/common-syntax).
- Use SANA enumerations for OBJECT_TYPE, TIME_SYSTEM, frames, and categories.

## Edge cases
- Use UNKNOWN for undisclosed identifiers or names where allowed.
- COUNTRY is unconstrained by the standard; establish coding in the ICD if needed.
- OCM_DATA_ELEMENTS is informational; do not rely on it for parsing.
