# Perturbations Specification

## Purpose and scope
This optional block declares the force‑model settings, sources, and fixed indices used to create trajectory and/or covariance content in the OCM. It enables receivers to reproduce propagation assumptions. At most one PERT block may appear in an OCM.

## Block keywords and order
Delimited by:
- PERT_START (line by itself)
- PERT_STOP (line by itself)

Within the block, KVN keys shall appear in the fixed order below.

## Validation and constraints

- Only one PERT block per OCM.
- GRAVITY_MODEL should include explicit degree/order if not intrinsic to the model name.
- If GM and/or EQUATORIAL_RADIUS override the model's values, ensure consistency across propagation and OD sections.
- Values listed as "fixed" override time‑varying inputs from SW_DATA_SOURCE; define precedence and usage in an ICD if ambiguity remains.
