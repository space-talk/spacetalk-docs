# Maneuver Specification

## Purpose and scope
A maneuver block describes a time history of propulsive activity or deployments. It supports impulsive ΔV, finite thrust (including duty cycles), direct acceleration profiles, and host/child deployment kinematics. Multiple blocks may appear (e.g., different devices, frames, or overlapping intervals).

## Block keywords and order
Delimited by:
- MAN_START (line by itself)
- MAN_STOP (line by itself)

Within the block, metadata KVN keys shall appear in the order below, followed by one or more non-KVN maneuver data lines defined by MAN_COMPOSITION, then MAN_STOP.

## Propulsive composition elements (Table 6‑8)

Choose from (names are normalized to the standard):
- TIME_ABSOLUTE or TIME_RELATIVE
- MAN_DURA (s)
- DELTA_MASS (kg; negative for mass loss)
- ACC_X, ACC_Y, ACC_Z (km/s**2)
- ACC_INTERP (ON/OFF)
- ACC_MAG_SIGMA (%) and ACC_DIR_SIGMA (deg)
- DV_X, DV_Y, DV_Z (km/s) — instantaneous at `Tstart + 0.5*MAN_DURA`
- DV_MAG_SIGMA (%) and DV_DIR_SIGMA (deg)
- THR_X, THR_Y, THR_Z (N)
- THR_EFFIC (η; 0–1)
- THR_INTERP (ON/OFF)
- THR_ISP (s)
- THR_MAG_SIGMA (%) and THR_DIR_SIGMA (deg)

Notes
- MAN_DURA = 0 denotes an impulsive maneuver.
- For finite burns, DELTA_MASS should reflect propellant use and any payload ejections during the interval (rocket equation consistency left to the producer).

## Deployment composition elements (Table 6‑9)

Choose from:
- TIME_ABSOLUTE or TIME_RELATIVE
- DEPLOY_ID (child identifier; "0" indicates no deployment at that line)
- DEPLOY_DV_X, DEPLOY_DV_Y, DEPLOY_DV_Z (child ΔV in MAN_REF_FRAME, km/s; applied at time tag)
- DEPLOY_MASS (kg; decrement to host, ≤ 0)
- DEPLOY_DV_SIGMA (%) and DEPLOY_DIR_SIGMA (deg)
- DEPLOY_DV_RATIO (dimensionless; host-to-child ΔV ratio; typically ≤ 0 to indicate opposite ΔV on host)
- DEPLOY_DV_CDA (m**2; child CD·A)

## Section rules and semantics

- All vectors are expressed in MAN_REF_FRAME (epoch MAN_FRAME_EPOCH if supplied).
- Blocks using DC controls:
  - If DC_TYPE = CONTINUOUS, no DC_* keys are required.
  - If DC_TYPE = TIME or TIME_AND_ANGLE, supply the required DC_* subset. DC_MIN_CYCLES/DC_MAX_CYCLES may override DC_EXEC_STOP.
- Interpolation of time-varying thrust/acceleration vectors (THR_INTERP/ACC_INTERP = ON) should use robust methods (e.g., Euler axis/angle; see Annex F).
- Only one spacecraft object is addressed per maneuver block. For deployments, MAN_DEVICE_ID should be DEPLOY, and the lines specify child ΔV and host mass decrement.
- Multiple blocks may share MAN_DEVICE_ID if time intervals do not overlap or represent different bases.

## Validation checklist

- Required keys present: MAN_ID, MAN_DEVICE_ID, MAN_REF_FRAME, DC_TYPE, MAN_COMPOSITION.
- If DC_TYPE ≠ CONTINUOUS, required DC_* keys supplied; if TIME_AND_ANGLE, supply angular triggers and frames.
- MAN_COMPOSITION elements are all from propulsive set or all from deployment set; includes exactly one time field (absolute or relative).
- Data-line element count matches MAN_COMPOSITION; timetag mode is consistent within the block.
- For ΔV impulses, apply at `Tstart + 0.5*MAN_DURA`. For mass changes: DELTA_MASS ≤ 0; DEPLOY_MASS ≤ 0; DEPLOY_DV_RATIO typically ≤ 0.
