# Orbit Determination (OD) Data

## Purpose and scope
This optional block summarizes the OD method, spans, data volumes, quality measures, and sensors associated with the orbit solution. At most one OD block may appear in an OCM. When IDs correspond, this block can be referenced by TRAJ_BASIS_ID, COV_BASIS_ID, and MAN_BASIS_ID.

## Block keywords and order
Delimited by:
- OD_START (line by itself)
- OD_STOP (line by itself)

Within the block, KVN keys shall appear in the fixed order below.

## Validation and constraints

- Only one OD block per OCM.
- OD_METHOD should indicate both algorithm class and tool to avoid ambiguity.
- OD_EPOCH must be interpretable using OCM TIME_SYSTEM (absolute or relative).
- Counts and spans are informational; consistency checks (e.g., ACTUAL_OD_SPAN vs. day differences) are recommended but not mandated.
